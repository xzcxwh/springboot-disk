注释格式规则总结：

所有注释都在代码上方一行

注释内容前添加序号+点号（如：1. 2. 3.）

注释紧跟在代码行前面，不需要多余的缩进空格

关键行可以添加简短的解释说明

不对头文件注释, 不对int mian()注释, 不对主函数的return 0;注释

需要对全局变量注释

//后面有一个空格, 序号后面没有空格

// 1.注释内容
代码行;

// 2.注释内容
代码行;

// 3.注释内容，可以有多行解释说明
// 这是第二行解释
代码行;
